# Postmortem Template

## Incident

Describe the failed connector, command, run ID, and user impact.

## Timeline

Record detection, investigation, mitigation, and resolution.

## Root Cause

Identify whether failure came from catalog metadata, runtime family logic, credential config, docs, query validation, write validation, or external API behavior.

## Corrective Actions

Add or update conformance tests, docs, runtime behavior, and runbook entries.
