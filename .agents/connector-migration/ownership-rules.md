# Connector CLI Surface Ownership Rules

These rules prevent shared-file collisions when many connector agents work in parallel. The
GitHub pilot established the shape; these rules generalize ownership for every connector.

## Coordinator-owned (shared) files

The parent orchestrator / coordinator owns these. A worker may not edit them unless the issue
explicitly grants scope:

- `.planning/STATE.md`, `.planning/ROADMAP.md`, `.planning/config.json`
- `.planning/phases/**/RUN-STATE.json`, `AGENT-ORCHESTRATION.json`, `PLAN.md` (parent-level)
- `.agents/agentic-delivery/**` (contracts, workflows, schemas, references) — docs-only changes
  require a dedicated docs issue, not a connector slice
- `internal/connectors/engine/**` (the declarative engine is shared infrastructure)
- `internal/connectors/hooks/**` (shared hooks; a connector-specific hook is worker-owned only
  when the issue assigns that connector and the hook lives under that connector's namespace)
- `website/lib/**`, `website/data/connectors.generated.json` (generated; regenerated by the
  coordinator or the worker that changed connector docs, never hand-edited)
- `internal/connectors/bundleregistry/registry.go` (generated bundle registry; regenerated, not hand-edited)
- `Makefile`, `.github/workflows/**`, `go.mod`/`go.sum` (human-gated)

## Worker-owned (connector-specific) files

A worker assigned connector `<name>` owns:

- `internal/connectors/defs/<name>/**` — `metadata.json`, `spec.json`, `streams.json`,
  `writes.json`, `api_surface.json`, `cli_surface.json`, `schemas/**`, `fixtures/**`
- `internal/connectors/native/<name>/**` — only if the connector needs Go (most connectors are
  declarative and have no native dir)
- `docs/connectors/<name>/**` — generated/validated connector docs
- `internal/connectors/native/<name>/*_test.go` — red-first package tests

## Rules

1. **One connector per worker.** A worker never edits another connector's dir.
2. **Generated files are regenerated, never hand-edited.** If `connectors.generated.json` drifts,
   run `pnpm run gen:website-data` and commit the output; do not patch the JSON by hand.
3. **Shared schema changes are their own issue.** Adding a new `execution_model` enum value or a
   new `record_schema` keyword is an engine change, not a connector slice — it needs its own
   issue and PR so it doesn't collide with in-flight connector work.
4. **Compound write hooks are connector-scoped.** A `hook: "<name>"` field routes to a
   connector's own hook package; do not add connector-specific compound-write logic to a shared
   hook owned by another connector.
5. **Branch and PR policy.** Worker branches start from the active rollout parent branch (e.g.
   `feat/44-github-cli-parity`), target that same parent branch, and use `Refs #<sub-issue>` and
   `Refs #44` — never closing keywords.
6. **Commits are coordinator-validated.** Workers push to their own branch; the coordinator owns
   merge validation, generated-file regeneration, and the parent-PR review-coverage record.
