SECURITY
  read risk: Crisp REST API reads may include website visitor, conversation, people, campaign, and helpdesk data. Direct-read commands are bounded to the configured byte limit and fixed official endpoints.
  write risk: Crisp REST API mutations can create, update, delete, import, export, or administer website resources. All write commands are explicit named reverse-ETL actions that require plan, preview, and approval; destructive/admin actions require typed confirmation.
  approval: Reverse ETL writes require plan, preview, approval token, and typed confirmation for destructive/admin operations.
  Never pass secret values in chat, shell arguments, logs, docs, or JSON output.
