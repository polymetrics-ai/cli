---
title: Installation
description: Install pm from a release binary, via go install, or by building the single Go binary from source.
---

`pm` ships as a single Go binary with no runtime dependencies in its default build. Pick the installation method that suits your environment.

<Callout type="info">
  **Prerequisites:** Go 1.25.11 or later is the only hard requirement for the default CGO-free build. Run `go version` to verify. For the DuckDB-enabled build you additionally need a C toolchain (gcc or clang).
</Callout>

## Installation methods

<Tabs items={["Release binary", "go install", "Build from source"]}>

<Tab value="Release binary">

Release assets are published from [`polymetrics-ai/cli`](https://github.com/polymetrics-ai/cli/releases) for Linux, macOS, and Windows on amd64 and arm64.

```bash
os_name="$(uname -s)"
arch_name="$(uname -m)"

case "$os_name" in
  Darwin) os=darwin ;;
  Linux) os=linux ;;
  MINGW*|MSYS*|CYGWIN*) os=windows ;;
  *) echo "unsupported OS: $os_name" >&2; exit 1 ;;
esac

case "$arch_name" in
  x86_64|amd64) arch=amd64 ;;
  arm64|aarch64) arch=arm64 ;;
  *) echo "unsupported architecture: $arch_name" >&2; exit 1 ;;
esac

gh release download --repo polymetrics-ai/cli --pattern "pm_*_${os}_${arch}.*"
case "$os" in windows) unzip "pm_"*_"${os}"_"${arch}".zip ;; *) tar -xzf "pm_"*_"${os}"_"${arch}".tar.gz ;; esac
binary=./pm
if [ "$os" = windows ]; then binary=./pm.exe; fi
"$binary" version
```

Each release includes `checksums.txt` for artifact verification.

</Tab>

<Tab value="go install">

Use this path when your environment can resolve the `polymetrics.ai` module path.

```bash
go install polymetrics.ai/cmd/pm@latest
```

The binary is placed at `$GOPATH/bin/pm`. Make sure `$GOPATH/bin` (or `$(go env GOPATH)/bin`) is on your `$PATH`.

To pin a specific version replace `latest` with a version tag, for example `v0.9.0`.

</Tab>

<Tab value="Build from source">

Clone the repository and build locally:

```bash
git clone https://github.com/polymetrics-ai/cli
cd cli
make build        # produces ./pm  (CGO-free, default)
make install      # copies ./pm to $GOPATH/bin
```

### DuckDB-enabled build

The default build uses a pure-Go JSONL warehouse. To unlock the full DuckDB SQL engine -- joins, window functions, aggregations -- build with CGO enabled:

```bash
CGO_ENABLED=1 go build -tags duckdb ./cmd/pm
```

<Callout type="warn">
  The DuckDB build requires CGO. Cross-compiling to a different OS or architecture also requires a matching C cross-toolchain. The default CGO-free build is recommended unless you specifically need analytical SQL.
</Callout>

</Tab>

</Tabs>

## Release cadence

Release PRs are generated from Conventional Commits after changes land on `main`. A connector update such as `fix(github): repair pagination` produces a patch release. A new connector such as `feat(connector): add linear` produces a minor release. Breaking changes use `!` or a `BREAKING CHANGE:` footer for a major release.

## Verify your installation

```bash
pm --help
```

```bash
pm connectors list | head -10
```

If you see a list of connector names, pm is installed and ready.
