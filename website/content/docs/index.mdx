`pm` is a single Go binary that extracts data from connector-backed APIs, lands it in an embedded DuckDB warehouse, lets you query it with real SQL, and writes approved results back through connector actions -- all without a vendor cloud, Docker, or Kubernetes.

## The extract-query-act loop

<ExtractQueryActDiagram />

Every step is accessible to a human operator and to an AI agent using identical commands.

## Key properties

| Property | What it means |
|---|---|
| **Local-first** | Credentials live in an encrypted local vault. Nothing reaches a vendor cloud unless you write it there. |
| **Agent-native** | Every command emits structured JSON via `--json`. Exit codes are stable and branchable. Writes require explicit approval. |
| **One binary** | `go install polymetrics.ai/cmd/pm@latest` and you're done. No daemon, no registry, no SaaS. |
| **Capability-based** | ETL streams and reverse-ETL actions are declared per connector, using the same credential store. |

## Top-level commands

```bash
pm init            # Initialize a local workspace
pm connectors      # Browse and inspect connector capabilities
pm credentials     # Manage the encrypted credential vault
pm connections     # Create and list data connections
pm catalog         # Refresh or show a connection's stream catalog
pm etl             # Run extractions and check status
pm query           # Run SQL over extracted data
pm reverse         # Plan, preview, and execute reverse-ETL writes
pm agent           # Generate agent plans
pm runtime         # Check runtime health (pm runtime doctor)
pm flow            # Orchestrate multi-step flows
pm extract         # Low-level extract command
pm perf            # Benchmarking utilities
pm docs            # Generate or validate connector docs
pm skills          # Generate connector-specific agent skills
pm rlm             # Run the RLM extractor agent
pm schedule        # Manage scheduled workflows
pm worker          # Run the Temporal worker
```

<Callout type="info">
  There is no `pm run` top-level command. Use `pm etl run` to trigger an extraction.
</Callout>
